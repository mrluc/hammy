remark

What is remark?

A simplistic, in-browser, markdown-driven slideshow tool targeted at people who know their way around HTML and CSS, featuring:

  • Markdown formatting, with smart extensions

  • Automatic syntax highlighting, with optional language hinting

  • Slide scaling, thus similar appearance on all devices / resolutions *

  • Touch support for smart phones and pads, i.e. swipe to navigate slides

* At least browsers are trying their best

Why use remark?

Since the slideshow is expressed using Markdown, you may:

  • Focus on the content, expressing yourself in next to plain text not worrying what flashy graphics and disturbing effects to put where

Since the slideshow is actually an HTML document, you may:

  • Display it in any decent browser

  • Style it using regular CSS, just like any other HTML content

Since the slideshow is contained in a file, you may:

  • Store it wherever you like; on your computer, hosted from your Dropbox, hosted on Github Pages alongside the stuff you're presenting...

  • Keep track of changes using your favourite SCM tool, like Git or Mercurial

How does it work?

A Markdown-formatted chunk of text contained in an HTML document is processed and separated into individual slides - all handled by JavaScript running in the browser.

 

HTML Document

A simple HTML document is needed for hosting the styles, markdown and the generated slides themselves: 

<!DOCTYPE html>
<html>
  <head>
    <script src="remark.js" type="text/javascript"></script>
    <style type="text/css">
      /* Slideshow styles */
    </style>
  </head>
  <body>
    <textarea id="source">
      <!-- Slideshow markdown -->
    </textarea>
    <div id="slideshow"><div>
  </body>
</html>

You may download remark.js to have your slideshow not depend on any online resources, or reference the latest version online directly.

Slideshow Markdown

The slideshow itself is expressed by a Markdown-formatted chunk of text: 

# Slide 1

This is the first slide.

---

# Slide 2

This is the second slide.

Regular Markdown rules apply with only a single exception:

  • A blank line followed by three dashes constitues a new slide
    (not a horizontal rule, <hr />)

Have a look at the Markdown website if you're new to markdown formatting.

Markdown Extensions

To help out with slide layout and formatting, a few extensions have been included:

  • Slide classes, for styling entire slides

  • Content classes, for styling specific content

  • Language classes, for syntax highlighting language hinting

Slide Classes

A line containing one or more dotted CSS class names is extracted, and the specified classes are applied to the current slide: 

.center.middle

# Slide with content centered in both dimensions

Resulting HTML extract: 

<div id="slideshow">
  <div class="slide">
    <div class="content center middle">
      <h1>Slide with content centered in both dimensions</h1>

The center and middle classes are included along with a few other common case classes.

Content Classes

Any occurences of one or more dotted CSS class names followed by square brackets are replaced with the contents of the brackets with the specified classes applied: 

.footnote[.red.bold[*] At least browsers try their best]

Resulting HTML extract: 

<span class="footnote">
  <span class="red bold">*</span> At least browsers try their best
</span>

Automatic Syntax Highlighting

Code blocks are automatically syntax highlighted, based on the number of recognized keywords from a set of supported languages.

If the language detected is incorrect, you may specify the language to use by adding a dotted language class on the first line of code: 

Code:

    .ruby
    def add(a, b)
      a + b
    end

Inline code is not automatically highlighted, and must be supplied a language class to be highlighted: 

Inline code: `.ruby a = 5`

To disable syntax highlighting, use the class no-highlight.

Syntax Highlighting Styles

A number of highlighting styles are available, including several well-known themes from different editors and IDEs.

You may specify a style other than the default style using the configuration option highlightStyle, like this: 

<script src="remark.js" language="text/javascript">
  { "highlightStyle": "sunburst" }
</script>

If you're using some custom style, you might want to prevent any of the bundled styles from being included. This can be done by specifying null as the style: 

{ "highlightStyle": null }
